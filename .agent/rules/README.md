# 프로젝트 개발 규칙

이 프로젝트는 다음 두 가지 핵심 규칙을 따릅니다:

## 1. TDD (Test-Driven Development)
코어 로직은 반드시 테스트 주도 개발 방식으로 구현합니다.

📖 자세한 내용: [tdd.md](./tdd.md)

**핵심 원칙:**
- 🔴 Red: 실패하는 테스트 먼저 작성
- 🟢 Green: 테스트를 통과하는 최소 코드 작성
- 🔵 Refactor: 코드 개선 및 리팩토링

**적용 대상:**
- ✅ 계산 엔진
- ✅ 상태 관리
- ✅ 히스토리 관리
- ✅ 유틸리티 함수

**제외 대상:**
- ❌ UI 컴포넌트
- ❌ 이벤트 핸들러
- ❌ DOM 조작

## 2. SOLID 원칙
모든 코어 로직은 SOLID 원칙을 준수하여 구현합니다.

📖 자세한 내용: [solid.md](./solid.md)

**5가지 원칙:**
1. **SRP** (Single Responsibility): 단일 책임
2. **OCP** (Open/Closed): 확장에 열림, 수정에 닫힘
3. **LSP** (Liskov Substitution): 하위 타입 치환 가능
4. **ISP** (Interface Segregation): 인터페이스 분리
5. **DIP** (Dependency Inversion): 의존성 역전

---

## 개발 워크플로우

```
1. 요구사항 분석
   ↓
2. 테스트 작성 (TDD - Red)
   ↓
3. SOLID 원칙을 고려한 설계
   ↓
4. 최소 구현 (TDD - Green)
   ↓
5. SOLID 원칙에 맞춰 리팩토링 (TDD - Refactor)
   ↓
6. 테스트 통과 확인
   ↓
7. 코드 리뷰 및 커밋
```

## 체크리스트

코드 작성 전:
- [ ] 테스트를 먼저 작성했는가?
- [ ] 각 클래스가 단일 책임을 가지는가?
- [ ] 확장 가능한 구조인가?
- [ ] 의존성 주입을 사용하는가?

코드 작성 후:
- [ ] 모든 테스트가 통과하는가?
- [ ] 커버리지가 80% 이상인가?
- [ ] SOLID 원칙을 위반하지 않았는가?
- [ ] 코드가 읽기 쉽고 유지보수 가능한가?

---

**이 규칙들은 프로젝트의 코드 품질과 유지보수성을 보장하기 위한 필수 사항입니다.**
