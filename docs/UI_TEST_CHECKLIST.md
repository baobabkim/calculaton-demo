# UI 수동 테스트 체크리스트

> 이 문서는 UI 기능을 수동으로 테스트하기 위한 체크리스트입니다.
> UI는 자동화된 테스트를 작성하지 않고, 이 체크리스트로 검증합니다.

## 테스트 환경

- [ ] 로컬 개발 서버 실행 (`npm run dev`)
- [ ] 브라우저: Chrome, Firefox, Safari, Edge
- [ ] 모바일 시뮬레이터 (Chrome DevTools)

---

## 1. 기본 계산 플로우

### 숫자 입력
- [ ] 0-9 버튼 클릭 시 디스플레이에 표시
- [ ] 연속 숫자 입력 시 올바르게 연결
- [ ] 소수점(.) 버튼 동작
- [ ] 소수점 중복 입력 방지

### 연산자
- [ ] + 버튼 클릭 시 수식에 추가
- [ ] - 버튼 클릭 시 수식에 추가
- [ ] × 버튼 클릭 시 수식에 추가
- [ ] ÷ 버튼 클릭 시 수식에 추가
- [ ] 연산자 연속 입력 시 마지막 것으로 교체

### 계산 실행
- [ ] = 버튼 클릭 시 결과 계산
- [ ] 결과가 디스플레이에 표시
- [ ] 수식이 상단에 표시

### 초기화 및 삭제
- [ ] AC 버튼 클릭 시 모든 입력 초기화
- [ ] 백스페이스 버튼 클릭 시 마지막 문자 삭제
- [ ] 빈 상태에서 백스페이스 클릭 시 오류 없음

### 괄호
- [ ] ( 버튼 클릭 시 수식에 추가
- [ ] ) 버튼 클릭 시 수식에 추가
- [ ] 괄호 우선순위 올바르게 계산
- [ ] 중첩 괄호 처리

## 2. 과학 함수

### 삼각 함수 (DEG 모드)
- [ ] sin(30) = 0.5
- [ ] cos(60) = 0.5
- [ ] tan(45) = 1
- [ ] DEG 모드 표시 확인

### 삼각 함수 (RAD 모드)
- [ ] DEG/RAD 토글 버튼 동작
- [ ] RAD 모드로 전환
- [ ] sin(π/2) ≈ 1
- [ ] cos(π) ≈ -1
- [ ] RAD 모드 표시 확인

### 로그 및 지수 함수
- [ ] ln(e) = 1
- [ ] log(100) = 2
- [ ] x² 버튼 동작 (예: 5² = 25)
- [ ] x³ 버튼 동작 (예: 2³ = 8)
- [ ] xʸ 버튼 동작 (예: 2^10 = 1024)
- [ ] √x 버튼 동작 (예: √16 = 4)

### 2nd 모드
- [ ] 2nd 버튼 클릭 시 모드 전환
- [ ] 버튼 레이블 변경 확인 (sin → sin⁻¹)
- [ ] sin⁻¹(0.5) = 30 (DEG)
- [ ] cos⁻¹(0.5) = 60 (DEG)
- [ ] 2nd 모드 해제 시 원래 함수로 복귀

### 기타 함수
- [ ] 1/x 버튼 동작 (예: 1/4 = 0.25)
- [ ] e 버튼 동작 (오일러 수)

- [ ] 개별 항목 삭제 버튼 동작

### 히스토리 영속성

## 6. 키보드 입력

### 숫자 및 연산자
- [ ] 0-9 키 입력
- [ ] + 키 입력
- [ ] - 키 입력
- [ ] * 키 입력 (곱셈)
- [ ] / 키 입력 (나눗셈)
- [ ] . 키 입력 (소수점)

### 특수 키
- [ ] Enter 키: = 버튼과 동일
- [ ] Backspace 키: 마지막 문자 삭제
- [ ] Escape 키: AC 버튼과 동일
- [ ] ( ) 키: 괄호 입력

### 포커스 관리
- [ ] Tab 키로 버튼 간 이동
- [ ] 포커스 표시 명확
- [ ] Enter 키로 포커스된 버튼 클릭

## 7. 오류 처리

### 계산 오류
- [ ] 0으로 나누기: "Error" 표시
- [ ] 음수의 제곱근: "Error" 표시
- [ ] 잘못된 수식: "Error" 표시
- [ ] 오버플로우: 과학적 표기법 또는 "Error"

### 입력 오류
- [ ] 빈 수식에서 = 클릭: 오류 또는 0 표시
- [ ] 연산자만 입력: 오류 처리
- [ ] 괄호 불일치: 오류 처리

## 8. 반응형 디자인

### 모바일 (< 768px)
- [ ] 레이아웃이 화면에 맞게 조정
- [ ] 버튼 크기가 터치하기 적절
- [ ] 스크롤 없이 모든 버튼 접근 가능
- [ ] 가로/세로 모드 모두 동작

### 태블릿 (768px - 1024px)
- [ ] 레이아웃이 화면에 맞게 조정
- [ ] 버튼 간격이 적절

### 데스크톱 (> 1024px)
- [ ] 레이아웃이 중앙 정렬
- [ ] 최대 너비 제한
- [ ] 마우스 호버 효과

## 9. 접근성

### 키보드 네비게이션
- [ ] 모든 기능을 키보드로 접근 가능
- [ ] Tab 순서가 논리적
- [ ] 포커스 트랩 없음

### 스크린 리더
- [ ] 버튼에 적절한 ARIA 레이블
- [ ] 디스플레이 영역이 live region으로 설정
- [ ] 상태 변경 시 알림

### 색상 대비
- [ ] 텍스트와 배경 대비 충분 (WCAG AA)
- [ ] 버튼과 배경 대비 충분
- [ ] 다크 모드에서도 대비 충족

## 10. 성능

### 로딩 시간
- [ ] 초기 페이지 로드 < 2초
- [ ] 모든 리소스 로드 완료 확인

### 반응성
- [ ] 버튼 클릭 즉시 반응
- [ ] 계산 결과 < 100ms
- [ ] 애니메이션 부드러움 (60fps)

### 메모리
- [ ] 장시간 사용 시 메모리 누수 없음
- [ ] 히스토리 50개 제한 확인
